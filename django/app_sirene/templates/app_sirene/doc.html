Sirene est un outil d'envoi de notifications à des utilisateurs  lors de la survenue d'événements de type incident, maintenance, crise, ..., sur des installations IT.

Canaux  de notification

Chaque message (notification) peut être diffusée de plusieurs façons :

  • Page publique : Internet, visible par tous. Page à choisir parmir les pages publiques préconfigurées.
  • Page privée: pour tous les utilisateurs authentifiés
  • Page privée personnelle: pour les utilisateurs authentifiés et sélectionnés en destinataires lors de l'envoi
  • Page privée anonyme (interne): pour les utilisateurs dont l'IP de connexion fait partie des IP de confiance (réseau interne)
  • Email : pour les destinataires sélectionnés lors de l'envoi (avec email configuré dans leur profil)
  • SMS: pour les destinataires sélectionnés lors de l'envoi (avec SMS configuré dans leur profil)

Pages publiques

Les pages publiques (tout Internet) contiennent une information minimale, non sensible, invitant l'utilisateur à se connecter pour consulter les notifications détaillées.

Elles sont prédéfinies. Il est possible en cas de situation particulière de fabriquer et publier une nouvelle page publique.

Messages prédéfinis

En majorité, les messages émis par les opérateurs de l'outil utilisent des messages préparés à l'avance choisis dans une liste. Un message prédéfini est constitué de :

  • Un titre : affiché dans les pages privées, et sert de contenu SMS / sujet d'EMAIL
  • Un corps de texte: en HTML dans les emails et sur les pages privées
  • Une sévérité  ou couleur : indicateur coloré visuel noir/rouge/orange/bleu/vert/gris, etc.
  • Une catégorie:  permet de grouper les messages p.ex incident / annonce / maintenanc, etc.
  • Un choix de page publique parmi les pages publiqes disponibles
  • Le choix de placer le message dans les messages privés accessibles à tous les utilisateurs authentifiés.
  • Le choix d'émettre un email pour les desinataires sélectionnés
  • Le choix d'émettre un SMS pour les destinataires sélectionnés
  • Des destinataires définis à l'aide de :
    • utilisateurs indivisuels ou membres de groupes d'utilisateurs
    • utilisateurs rattachés à des sites ou à des groupes de sites
    • utilisateurs rattachés à des applications (gestionnaire, infogérant, etc.)

Règles de notifications

Chaque utilisateur ne reçoit qu'une fois un email ou un SMS, même s'il appartient à plusieurs groupes, sites.

Un utilisateur peut configurer ses préférences pour recevoir ou pas les notifications individuelles,  par email, par sms, configurer un email ou numéro de mobile secondaire, choisir s'il veut des notifications en 24/7 ou en heures ouvrées uniquement, etc.

Un message destiné  à un groupe d'utilisateur:

  • sera distribué à tous les utilisateurs du groupe d'utilisateur
  • sera distribué également à tous les utilisateurs des groupes inclus dans ce groupe (hiérarchie des groupes)

Un message destiné  à un site ou à un groupe de site:

  • sera distribué à tous les utilisateurs configurés comme rattachés au site ou à l'un des sites du groupe de site
  • distribution Email, SMS, ou affichage en page privée selon choix de l'émetteur
  • Exemple : le site XXX subit une panne réseau

Un message adressé à une application:

  • sera distribué au utilisateurs attachés aux applications (les gestionnaires , mainteneurs)
  • sera distribué aux sites (utilisateurs des sites) ayant une configuration indiquant utiliser cette application (les utilisateurs)
  • distribution Email, SMS, ou affichage en page privée selon choix de l'émetteur
  • Exemple: l'application XXX est en panne

Un message ayant une page privée:

  • sera visible par tous les utilisateurs authentifiés
  • sera visible par tous les utilisateurs se connectant avec une IP de confiance (paramétrage de l'outil)

Configuration d'un Site

Un site (établissement, lieu physique, client) est décrit par:

  • un nom unique (clé d'identification, court, sans caractères spéciaux)
  • un nom d'affichage
  • une description
  • des utilisateurs "rattachés" à ce site (à prévenir en cas de notification destinée à ce site)
  • des applications utilisées par ce site ; les utilisateurs du site sont notifiés lorsque ces applications le sont

Configuration d'une Application

Une application est décrite par:

  • un nom court unique
  • un nom d'affichage
  • une description
  • un site d'hébergement indicatif (aucun usage pour de la notification !)
  • un ensemble d'utilisateurs ou groupes gestionnaires de cette application (à prévenir en cas de notification destinée à cette application);

IMPORTANT : les utilisateurs d'une application devant être notifiés, ne sont pas rattachés directement dans la configuration de l'application. Ils sont rattachés à des sites. Les sites décrivent les applications qu'ils utilisent.

Configuration d'un Groupe d'utilisateurs

  • les groupes d'utilisateurs peuvent contenir d'autres groupes d'utilisateurs
  • il convient de créer des arborescences d'organisation et placer les utilisateurs uniquement dans les groupes d'extrémité de l'arborescence.

Rôles utilisateurs

Un utilisateur authentifié peut avoir un ou plusieurs rôles, lui autorisant un ensemble de gestes dans l'outil :

  • contact: utilisateur n'ayant pas de login, mais pouvant recevoir des mails/sms ; exemple: partenaire, liste de diffusion
  • visiteur : utilisateur authentifié, mais sans déclaration ni existence dans l'application ; n'est pas assignable à des sites ou applications. Peut consulter les messages en pages privées. Ne recevra pas de SMS/Email.
  • opérateurs: émettent des messages mais ne modifient pas la configuration de l'outil (structure, messages)
  • administrateurs locaux: gèrent les applications, groupes et  sites existants. Ajoutent des utilisateurs, les rattachent
  • administrateurs globaux: créent les structures, les messages prédéfinis. Attribuent les rôles.
  • administrateur système : gèrent le paramétrage technique de l'application

Les permissions fines associées à chaque rôle sont paramétrables et de nouveaux rôles peuvent être créés pour des cas d'usages ou populations particulières.

Messages prédéfinis génériques

Certains messages prédéfinis peuvent être rendus génériques pour s'appliquer à un/des sites, ou une/des applications choisie au moment de l'envoi. L'opérateur devra alors nécessairement sélectionner à chaque envoir les sites ou applications concernées. Le motifs suivants sont disponibles dans le corps des messages;
  • $APPS$
  • $SITES$

Ils sont remplacés par la liste des applications (respectivement la liste des sites) sélectionnés par l'opérateur au moment de l'envoi.

Délai d'envoi, volumes, quota

A l'envoi, les messages sont mis en file d'attente puis transmis aussi rapidement que possible. Un délai de quelques secondes ou plus peut se rencontrer.

Pour des raisons de coûts, des quota de SMS sont implémentés et configurables (quantité / jour / opérateur).

Scopes

Les scopes désignent un "ensemble  de destinataires" et sont utilisés  dans l'outil pour définir les cibles de notifications quelle que soit leur type (site, app, groupe, ...).

Scopes disponibles:

  • user:xxxx  - utilisateur individuel  (login ou contact)
  • usergroup:xxxx  -  groupe d'utilisateur
  • site:xxxx - un site particulier
  • sitegroup:xxxx - un groupe de site
  • app:xxxx - une application (les utilisateurs attachés à cette application)
